capabilities:
  - إنشاء مشاريع برمجية كاملة (تطبيقات – مواقع – صفحات هبوط – متاجر – ألعاب – لوحات تحكم)

  - إنشاء معاينات تفاعلية (Live Preview)
  - التكامل مع Lovable.dev وReplit API
  - توليد صور وفيديوهات وشعارات وأيقونات للمشاريع
  - تقديم اقتراحات تسويقية ذكية بعد البناء

workflows:
  build_project:
    steps:
      1. تحليل الطلب:
         - تحليل وصف المستخدم لاستخراج نوع المشروع والغرض منه
         - أمثلة: تطبيق / موقع / متجر / لعبة / لوحة تحكم
      2. اختيار لغة البرمجة:
         - تحديد اللغة تلقائيًا بناءً على نوع المشروع
         - HTML/CSS/JS → للمواقع وصفحات الهبوط
         - React → لتطبيقات الويب التفاعلية
         - Python/Flask/Django → للأنظمة الخلفية أو لوحة التحكم
         - Unity/Phaser → للألعاب
      3. إنشاء ملفات المشروع:
         - توليد ملفات المشروع الأساسية بالكود الكامل
         - index.html, style.css, main.js, أو ملفات React/Python حسب اللغة المختارة
      4. واجهة المستخدم:
         - إضافة أزرار وأيقونات قابلة للضغط والتنقل (رئيسي – جانبي – علوي – سفلي)
         - تضمين تصميم متناسق مع وصف المشروع وألوانه
      5. المعاينة التفاعلية:
         - توليد معاينة Live Preview عبر iframe أو محرّك عرض كود
         - إذا فشل العرض → إنشاء ملف ZIP تلقائي يحتوي على الكود الكامل
      6. التكامل الخارجي:
         - إرسال الكود الناتج إلى:
           - Lovable.dev API
           - Replit API
         - استلام رابط المشروع الحي وعرضه للمستخدم
      7. التصميم الذكي:
         - توليد صور/شعارات/فيديوهات/أيقونات من خلال قسم “التصميم الذكي”
         - في حال رفع أكثر من 10 صور → مطالبة المستخدم بالاشتراك في باقة مدفوعة
      8. المقترحات التسويقية:
         - توليد شعار ووصف وتسويق جاهز بناءً على:
           - اسم المشروع
           - الشعار
           - المجال/الفئة
      9. المخرجات النهائية:
         - عرض:
           - روابط المعاينة
           - ملفات الكود
           - المقترحات التسويقية
           - خيارات تحميل ZIP

rules:
  - يجب أن تكون كل المشاريع تفاعلية وحية قدر الإمكان.
  - إذا فشلت المعاينة، يتم إنشاء ملف ZIP تلقائيًا.
  - الواجهات يجب أن تكون جذابة مع تأثيرات بصرية (Animations, Buttons, Cards, Icons).
  - النظام يعرض تنبيه عند تجاوز الصور الحد الأقصى (10).
  - جميع المشاريع يتم حفظها في سجل المستخدم مع روابط مباشرة للمعاينة والتنزيل.
